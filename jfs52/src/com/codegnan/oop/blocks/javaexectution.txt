Java Execution flow:
--------------------

step1: Write the program
step2: Compilation (javac) , .class is generated
step3: 'java' command will begin the JVM Execution .
       a. JVM will load .class into memory, this is called Class Loading
          Types of class loaders: 
            - Bootstrap classloader:
              Loads core java classes like java.lang.* from JDK's
              internal libraries.
            - Extension classloader:
               Loads classes from jdk extension directories like lib/ext
               for optional packages
            - Application class loader:
              Loads classes from the application class path(class path
              is where your project classes and external libraries are kept)
            
         b. Linking Phase -- Jvm after loading classes will links the class
               Linkig phase will havve the following stages: 
               
               	-Verification phase
                   Bytecode will be checked
                -Preparation phase: 
                 Memory is allocated for static variables
                 Default values assigned
                -Resolution stage:
                   Reference variables will refer to actual aobjects
          c. Initialization phase: 
               - the actual values are assigned 
                 - first static variables are initialized
                  -next, static blocks are executed
                  
                the static variable initilization and static block
                execution happens only once
                
                 
        d. main() method execution
            - Whenever object is created: 
               a. instance variables are initialized
               b. instance block executed
               c. Constructor executed
        e. Method execution happens
        f. main() completes 
        g. JVM stops       
        
        Memory areas used by JVM: 
        =========================
        Method Area:Stores class info, static variable
        Heap:  Objects
        Stack - object references, method calls.
        PC Registers: current instruction 
        
        
        Static Block vs Instance initializer block: 
        ============================================
        
        Static block belongs to class
        IIB(instance initializer block) block  belongs to object. 
        
        Static block runs only once at the begining
        Instance block runs for every object
        
        static block executes while class loading. 
        instance block executes whileobject creation
        
        static blocks can be used to perform static variable initialization
        and perform some complex operations for static data
        instance blocks are generally used to perform common constructor
        logic. 
        
        static blocks can access only static members directly
        instance blocks can can access both static and instance mebers
